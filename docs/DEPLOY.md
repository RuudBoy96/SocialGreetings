# Deploy SocialGreetings

## Quick test URL (temporary)

While your PC is running the app, expose it with Cloudflare Tunnel (no account needed):

```powershell
cd "path\to\Test 1"
$env:FLASK_DEBUG = "false"
.\.venv\Scripts\python.exe app.py

# In a second terminal:
.\bin\cloudflared.exe tunnel --url http://127.0.0.1:5000
```

Copy the `https://….trycloudflare.com` URL from the tunnel output. The URL changes each time you restart the tunnel.

For Stripe webhooks or Prodigi print assets, set in `.env`:

```env
PUBLIC_BASE_URL=https://your-tunnel-url.trycloudflare.com
```

## Permanent hosting (Render — free tier)

The repo includes `Dockerfile` and `render.yaml` for [Render](https://render.com).

1. Sign in at [dashboard.render.com](https://dashboard.render.com) with GitHub
2. **New → Blueprint** → select **RuudBoy96/SocialGreetings**
3. After deploy, copy your service URL (e.g. `https://socialgreetings.onrender.com`)
4. In Render **Environment**, set:
   - `PUBLIC_BASE_URL` = your Render URL
   - `SECRET_KEY` and `CARD_DATA_KEY` (auto-generated by blueprint)
   - Optional: `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `PRODIGI_API_KEY`
5. Stripe webhook endpoint: `https://your-app.onrender.com/webhooks/stripe`

First deploy may take several minutes (Playwright + Chromium in Docker). Free tier sleeps after inactivity — first visit may be slow to wake.
